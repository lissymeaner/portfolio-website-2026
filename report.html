Reflecting on my module experience

This document details my experience with the Web Development (a.k.a. web dev) module so far - discussing the highlights and notable challenges I faced while learning the practical skills of HTML and CSS, as well as my design decisions.

How am I finding the module?

It has been easy. With my 3 years of web dev experience, I understand most concepts taught in the module.

Positives

In weeks 1 and 2, I planned the website development using a content brief for my text/media (fig. 1.1) and a Gantt chart (fig. 1.2). This allowed me to avoid placeholder text early while coding my final site.

I enjoyed creating the responsive layout, as I could experiment with different solutions. I used a hybrid way: display:grid; on desktop (see fig. 2), and display:flex; on mobile. Choosing flexbox on mobile allowed the semantic elements (aside, nav, header, main, footer) to be stacked vertically in a simple yet flexible manner.

Developer tools from MS Edge and VSCode (see fig. 2) were well relied on when debugging CSS issues and testing the responsive layout. This approach came with almost no AI assistance (except where noted), which boosted my confidence working on this assignment.

One standout feature was the country selection dropdown with all countries (recognised by the United Nations), which was AI-automated to avoid manually typing a large number of option elements.

Challenge

A major challenge during development was implementing the submenu animation for the desktop navbar - the text would not disappear on time once the submenu was hovered off. This was solved by replacing the opacity transition with another transition for the left property.

Design decisions

I began developing my website in Week 3, focusing on coding the layout, mobile optimisation, and colour styling. The design overall was influenced by a few portfolio websites.

Jeznach's fixed left sidebar layout was chosen for the desktop design for easier navigation on wider screens. This decision also justified using an otherwise rarely-used tag like <aside>. The navbar, like Jeznach's, also works differently on mobile: instead of the sidebar, visitors see a full-screen overlay navigation.

Originally inspired by Matt Williams, I imagined my portfolio with dotted star animations around the homepage. He used JavaScript for his background, though, so I couldn't do as much here. Instead, I opted for autoplaying full-sized video backgrounds - a more convenient method. For this feature, I borrowed existing code from a blog post by CSS Tricks' author. I modified his method by adding two videos - each has a separate class that alternates between display:none; and display:block; based on the chosen system colour scheme.

To create an inverse of Farley's colour scheme, I made a green and black colour scheme in 'dark mode'. Most of the colours changed are stored in :root variables, so no separate file is needed. Some elements need an entirely different colour variable, though, which is why I kept a 'dark.css' file.
